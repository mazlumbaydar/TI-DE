# ============================================================
# Threat  : Axios npm Supply Chain — Cross-Platform RAT Dropper
# Platform: VMware Carbon Black CBC (Cloud)
# TTPs    : T1195.001 / T1059.001 / T1059.004 / T1059.006 / T1036.005
# Severity: Critical
# Rules   : 2 consolidated (process spawn + process-from-staging-path)
# Syntax  : CBC Lucene. All field values lowercase in index.
# Note    : CBC does NOT provide general file creation events.
#           File-drop detection requires CBC Audit & Remediation (LiveResponse).
#           Rule 2 detects execution FROM staging path (post-drop signal).
# ============================================================

# ── RULE 1: Package Manager Spawning Network or Scripting Tool ──
# Covers Linux and Windows child process types in one query.
# parent_name = npm or node; childproc_name = network/scripting tool.
# Values must be lowercase — CBC indexes are case-insensitive via lowercase.
(parent_name:node OR parent_name:npm OR parent_name:node.exe)
AND childproc_name:(curl OR wget OR python3 OR python OR bash OR sh OR nc OR ncat
                    OR wscript.exe OR cscript.exe OR powershell.exe OR pwsh.exe)

# ── RULE 2: Suspicious Execution from OS Staging Paths ──
# CBC cannot detect file creation events directly (no FileCreatedEvent).
# Instead, detect execution OF a process from the staging path.
# Any process running from ProgramData (not msiexec-installed) is suspicious.
# Linux /tmp execution also covered via process_path pattern.
(process_path:c\:\\programdata\\* AND process_name:*.exe AND NOT parent_name:msiexec.exe)
OR (process_path:/tmp/* AND process_name:(*.py OR *.sh))
