Background

What Happened

On 31 March 2026, two malicious versions of the axios npm package were identified: axios@1.14.1 and axios@0.30.4. The legitimate axios version line is 1.7.x — version 1.14.1 has never existed in the official package, making it an immediate red flag for version-aware teams.

Both packages contained a postinstall hook in package.json pointing to node ./scripts/install.js. The install.js file used XOR obfuscation to conceal C2 URLs and binary download logic. Upon execution, it detected the host operating system and downloaded a platform-specific RAT binary from sfrclak.com.

The RAT deployed differently per platform: on Windows it placed an .exe in %TEMP%\ and installed a scheduled task named "WindowsUpdate"; on Linux / macOS it wrote an ELF or Mach-O binary to ~/.local/bin/ and created a crontab entry running every 5 minutes. All variants beaconed to sfrclak.com:8000 using a fake Internet Explorer 8 User-Agent string (Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 6.1)) — an impossible string on any modern system, making it a zero-false-positive detection signal.

If you have axios@1.14.1 or axios@0.30.4 installed, uninstall immediately and treat the host as compromised. Run npm list axios to check all workstations and CI/CD environments.

Attack Flow

Attack Chain — 5 Steps

🎭
Step 1 — Package Publishing
Threat actor publishes axios@1.14.1 and axios@0.30.4 to npm. Uses version numbers that don't exist in legitimate axios (real versions are 1.7.x). Package contains malicious scripts/install.js and XOR-encoded payload. The inflated version number (1.14.1 > 1.7.x) can fool automated dependency update tools.
T1195.001 — Compromise Software Dependencies and Development Tools
📦
Step 2 — npm install / Hook Triggered
Developer or CI/CD pipeline runs npm install. npm automatically runs postinstall scripts. install.js executes with full Node.js capabilities — no user prompt, no warning, no permission check. The user sees only normal install output while the RAT dropper runs silently.
T1059.007 — JavaScript T1195.001
💻
Step 3 — OS Fingerprinting + Binary Download
install.js checks process.platform. Downloads OS-specific RAT binary from sfrclak.com via HTTPS. XOR-decodes the payload on-disk before execution. Binary is platform-native: .exe (Windows), ELF (Linux), Mach-O (macOS). The HTTPS download uses the system's CA store and looks like normal package telemetry.
T1059.007 — JavaScript T1140 — Deobfuscate/Decode Files
🔁
Step 4 — Persistence Installation
Windows: scheduled task named "WindowsUpdate" running every 5 minutes — deliberately chosen to blend with Microsoft's legitimate Windows Update tasks. Linux/macOS: crontab entry (*/5 * * * *) for the current user. RAT process is renamed to mimic a legitimate system utility name in process listings.
T1053.005 — Scheduled Task T1053.003 — Cron T1036 — Masquerading
📡
Step 5 — RAT C2 Beacon + Data Theft
RAT connects to sfrclak.com:8000 using fake MSIE 8.0 User-Agent — an IE8 string from 2009, impossible on any modern system. Exfiltrates environment variables, SSH keys (~/.ssh/id_rsa), .npmrc auth tokens, and git credentials. Attacker gains persistent remote access to the developer's machine and potentially the entire organization's infrastructure.
T1071.001 — Web Protocols T1001 — Data Obfuscation T1552.001 T1041 — Exfiltration over C2

Execution
JavaScript Postinstall Hook — Automatic Execution on npm install
T1059.007
Technique: Command and Scripting Interpreter: JavaScript. The postinstall hook in package.json runs node ./scripts/install.js automatically when npm resolves the dependency. No user interaction beyond npm install is required — the hook fires silently.
Evidence Artifacts npm verbose log / process tree
npm verbose log showing: "> axios@1.14.1 postinstall" "> node ./scripts/install.js" node.exe spawned by npm.cmd — parent-child process tree network connection from node.exe to sfrclak.com within 3s of install
Detection Note: "postinstall script runs immediately on npm install — no additional user action needed." Flag any node.exe making outbound connections during npm install to non-registry hosts.
Defense Evasion
Triple Evasion: XOR Obfuscation + Process Masquerade + Antique UA
T1140 T1036 T1001
T1140: XOR decoding of payload URLs and binary content on-disk defeats simple string-based YARA/AV signatures. T1036: RAT binary renamed to mimic system utilities (svchost_update.exe on Windows, update on Linux). T1001: MSIE 8.0 User-Agent in C2 traffic disguises beacon as legacy browser activity.
Evidence Artifacts Proxy logs / filesystem / memory
XOR-decoded binary written to %TEMP%\svchost_update.exe process renamed to system utility name in tasklist/ps output MSIE 8.0 UA string in proxy/firewall logs: "Mozilla/4.0 (compatible; MSIE 8.0; Windows NT 6.1; Trident/4.0)"
Detection Note: "Triple evasion: XOR on download, system name masquerade, antique UA string for C2 traffic." The MSIE 8.0 UA is a zero-false-positive signal — no modern application legitimately sends this string.
Persistence
Scheduled Task (Windows) + Cron Job (Linux/macOS)
T1053.005 T1053.003
T1053.005: Scheduled task named "WindowsUpdate" created via schtasks /create running every 5 minutes, deliberately chosen to blend with Microsoft's legitimate Windows Update task. T1053.003: crontab entry added for the current user on Linux/macOS with */5 * * * * frequency. Both mechanisms survive reboots and are installed without admin/root privileges.
Evidence Artifacts schtasks / crontab / Windows Event Log 4698
Windows: schtasks /query /tn "WindowsUpdate" → task pointing to %TEMP%\svchost_update.exe Event ID 4698 — Task created by node.exe (NOT by Microsoft) Linux/macOS: crontab -l → "*/5 * * * * ~/.local/bin/update" process restart observed after manual kill
Detection Note: "Task named WindowsUpdate: deliberate confusion with Microsoft's legitimate task name." Alert on WindowsUpdate scheduled task where the author is NOT NT AUTHORITY\SYSTEM or a Microsoft process.
Collection + Exfiltration
Credential Harvest + C2 Exfiltration over HTTP
T1552.001 T1005 T1041 T1071.001
T1552.001: SSH private keys (~/.ssh/id_rsa), npm auth tokens (~/.npmrc), git credentials (~/.gitconfig), and environment variables containing TOKEN, KEY, SECRET, or PASS are harvested. T1041: All data exfiltrated to sfrclak.com:8000 via HTTP POST, with beacon traffic disguised using MSIE 8.0 UA (T1071.001).
Evidence Artifacts File access audit / proxy logs / SIEM

Detection Note: "SSH key access by non-ssh process is zero-FP on developer machines — highest priority alert." Create an alert for any process reading ~/.ssh/id_rsa where process.name != ssh and process.name != gpg-agent.

Incident Response

Response Actions

🗑
Remove Malicious Packages Immediately
Run npm uninstall axios on all affected hosts, then reinstall a verified clean version: npm install axios@1.7.9. Audit package-lock.json across all repositories for any reference to axios@1.14.1 or axios@0.30.4.
🔤
Kill RAT + Remove Persistence
Windows: schtasks /delete /tn "WindowsUpdate" /f and delete %TEMP%\svchost_update.exe. Linux/macOS: crontab -e to remove the */5 * * * * entry, then delete ~/.local/bin/update. Kill any running RAT process.
🔑
Rotate All Secrets
Immediately rotate: SSH keypairs (~/.ssh/id_rsa), npm auth tokens (~/.npmrc and npm registry), git credentials (~/.gitconfig), and any environment variables containing TOKEN, KEY, SECRET, or PASS. Treat all credentials as compromised.
🔒
Block C2 at Network Level
Add firewall and DNS block rules for sfrclak.com and all subdomains. Block IP 142.11.206.73 at perimeter firewall. Add proxy block rule for any User-Agent containing MSIE 8.0 or Trident/4.0 from internal hosts.
🔎
Hunt for Beacons in Proxy Logs
Search historical proxy and firewall logs for MSIE 8.0 User-Agent from internal IPs. This pattern has zero false positives on modern enterprise networks. Any match indicates active or past C2 beaconing — escalate immediately to full IR.
📊
Audit npm Package Versions
Run npm list axios --all across all developer workstations and CI/CD environments. Remember: 1.14.1 does not exist in legitimate axios — any environment showing this version is compromised. Enforce npm ci with package-lock.json integrity verification in all pipelines.
