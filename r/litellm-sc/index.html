Background

What Happened

On 26 March 2026, the threat actor group TeamPCP uploaded a malicious version of the litellm Python package to PyPI (CVE-2026-33634). LiteLLM is a widely used open-source library that provides a unified interface to 100+ AI providers (OpenAI, Anthropic, Google, AWS Bedrock, etc.), making it a high-value target for credential theft in AI/ML environments.

The malicious package used a sophisticated .pth file persistence mechanism: during installation, a .pth file is written into Python's site-packages directory. Python automatically executes all .pth files at interpreter startup — meaning the malicious code runs on every Python invocation, not just when litellm is imported. The payload harvested API keys from environment variables and from common AI configuration files, exfiltrating them to models.litellm.cloud — a convincing typosquat of the legitimate LiteLLM infrastructure.

If litellm was installed between 24–26 March 2026, rotate ALL AI API keys immediately. The .pth file may persist even after uninstalling litellm. Check: find $(python -c "import site; print(site.getsitepackages()[0])") -name "*.pth" -newer /tmp

Attack Flow

Attack Chain — 6 Steps

📤
Step 1 — Malicious Package Uploaded to PyPI
TeamPCP creates and uploads a malicious litellm package to PyPI. The package closely mirrors legitimate LiteLLM metadata to pass automated PyPI checks. A .pth file is embedded that will be written to Python's site-packages at install time. The C2 domain models.litellm.cloud closely resembles the legitimate litellm.ai domain.
T1195.001 — Compromise Software Dependencies
📥
Step 2 — pip install Writes .pth File
Developer runs pip install litellm. During installation, setup.py writes sitecustomize.pth to Python's site-packages directory. This is a standard Python feature: .pth files in site-packages are read by the Python interpreter at every startup. No elevated privileges required — user-level pip install is sufficient.
T1059.006 — Python T1546 — Event Triggered Execution
⚙
Step 3 — Every Python Invocation Triggers Exfil
Every time Python starts — scripts, jupyter, pytest, FastAPI, celery workers — the .pth file executes the malicious payload. The payload reads API keys from environment variables and configuration files. Because this happens at interpreter startup, the credential theft occurs even in isolated virtual environments if the .pth was placed in the base Python installation.
T1546.004 — .bash_profile / .pth Persistence T1552.001 — Credentials in Files
🔑
Step 4 — AI API Key Harvest
Payload harvests OPENAI_API_KEY, ANTHROPIC_API_KEY, COHERE_API_KEY, AWS_ACCESS_KEY_ID/SECRET_ACCESS_KEY, GOOGLE_API_KEY, and any keys matching common AI API patterns from environment variables. Additionally reads ~/.config/litellm/config.yaml, .env files in the current working directory, and ~/.litellm.env.
T1552.001 — Credentials in Files T1083 — File and Directory Discovery
🖥
Step 5 — systemd Persistence for Long-Running Exfil
On Linux systems with systemd, the payload drops a user-level systemd service unit that runs a background Python process for continuous monitoring and exfiltration. This persists across reboots and runs independently of any Python scripts the developer runs, enabling ongoing credential surveillance even after the malicious package is removed.
T1053.006 — Systemd Service T1543 — Create or Modify System Process
📡
Step 6 — C2 Beacon — AI Keys Exfiltrated
Harvested credentials are sent via HTTPS POST to models.litellm.cloud (port 443) with a JSON payload resembling legitimate LiteLLM telemetry. The traffic blends with legitimate LiteLLM API calls. Secondary C2 via checkmarx.zone is used for exfiltration if the primary is blocked. Stolen keys enable attackers to run arbitrary AI workloads at the victim's expense.
T1041 — Exfiltration over C2 T1071.001 — Web Protocols (HTTPS)

Detection Engineering

Detection Rules — 10 Platforms

All rules are TTP-based and behavioral. Key detection signals: Python reading .env files at startup (not import time), unexpected outbound connections from Python processes to non-standard ports, new .pth files written to site-packages by pip.
