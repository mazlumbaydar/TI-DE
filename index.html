Engineering Standards

Detection Methodology

🎯
Rule 01
TTP Over IOC
Never hardcoded IPs or domains. Attacker infrastructure rotates in hours; adversary TTPs persist for years.
🔗
Rule 02
Min. 2 AND Conditions
Every rule requires at least 2 correlated conditions. Single-signal detections produce excessive noise.
📐
Rule 03
Max 2 Rules / Threat
Rule 1 targets process chain. Rule 2 targets artifact/file activity. Consolidation prevents alert fatigue.
✅
Rule 04
Platform Field Verified
All field names verified against official docs. Cortex XDR rules are BIOC-compatible — no joins, one event_type per rule.
⚡
Rule 05
KEV Priority
CISA KEV entries and actively exploited CVEs processed within 24 hours of confirmed exploitation intelligence.
🌍
Rule 06
Open Source · TLP:WHITE
All detection rules freely available on GitHub. No paywalls. The community grows stronger when detection logic is shared openly.
🔗Parent-Child Process Correlation
Target the spawn relationship, not just the child. langflow → bash is a detection; bash alone is noise.
🚫No Specific Filenames as IOCs
Not wt.exe — the signal is writer process + staging directory. Attackers rename trivially; path patterns are durable.
📌MITRE ATT&CK Mapped
Every rule maps to ≥1 MITRE technique. Enables correlation with actor profiles and coverage gap analysis.
🛡️BIOC-Compatible XDR Rules
Cortex XDR rules use xdr_data with single event_type per rule — no joins. Ready to import as Behavioral Indicators.
